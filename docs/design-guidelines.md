# ClaimEasy Design Guidelines

ClaimEasy follows the provided teal browser-window reference without copying its content.

- Mood: friendly insurance desk inside a polished desktop app window.
- Palette: teal gradient background, seafoam lower glow, white text in the hero, coral-pink primary action for claim operations, dark teal panels for contrast.
- Layout: a large rounded browser shell with traffic-light controls, tabs, address pill, floating outline shapes, central app icon, and a compact claim console below the hero.
- UX difference from previous projects: no long stacked intake form. Inputs live as editable claim chips, evidence reels, and transaction command buttons.
- Scroll rule: only the browser/document scrollbar is allowed; no inner page scroll container.
